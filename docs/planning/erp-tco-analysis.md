# ERP 选型成本收益与 TCO 分析

> 对应框架 Agent E：当前成本清单、ERPNext 自部署成本、迁移/培训/升级维护成本、3 年 TCO 对比。

## 1. 当前成本基线

### 1.1 已知成本（用户提供）

| 成本项 | 当前金额（年） | 说明 |
|--------|---------------|------|
| 用友软件及维护 | ¥2,500 - 数万 | 目前用友费用区间，低值可能为单账套基础版，高值可能为多公司/模块扩展 |
| 云服务/服务器 | ¥数千 - ¥10,000 | 现有 Workspace 部署所用云资源 |
| **已知年成本合计** | **约 ¥5,000 - ¥50,000+** | 取中位数约 ¥2-3 万/年 |

### 1.2 隐性成本（需估算）

| 成本项 | 估算依据 | 年成本 |
|--------|----------|--------|
| Excel 人工汇总与重复录入 | 财务人员跨系统复制数据工时 | 未量化，但随公司规模增长而放大 |
| 多系统数据对账 | 用友、Excel、Workspace 数据不一致时的人工核对 | 未量化，月度/季度集中发生时成本高 |
| 自研财务引擎维护 | Workspace 总账/报表/重分类系统的持续开发 | 已计入开发人力，未单独核算 |

**关键观察**：当前成本结构里，软件许可费不高，但隐性的人工重复劳动和数据对账成本未显性化。ERP 选型的主要收益不在"省掉用友许可费"，而在"减少重复劳动、统一数据源"。

## 2. ERPNext 自部署成本估算

### 2.1 软件许可成本

| 项目 | ERPNext | 说明 |
|------|---------|------|
| 软件许可 | **¥0** | GPLv3 开源，无用户限制，无模块限制 |
| 对比用友 | 省 ¥2,500 - 数万/年 | 但财务合规验证通过前不能承诺此项节省 |

### 2.2 基础设施成本

| 部署方式 | 月费用 | 年费用 | 适用场景 |
|----------|--------|--------|----------|
| 云 VPS 自部署（基础：4C8G） | $20-50 | ¥1,700-4,200 | 中小规模，技术团队可维护 |
| 云 VPS 自部署（标准：8C16G） | $50-100 | ¥4,200-8,400 | 多公司、数据量大 |
| 云 VPS 自部署（高配：16C32G） | $100-200 | ¥8,400-16,800 | 大规模或高并发 |
| Frappe Cloud 托管（共享） | $25 起 | ¥2,100 起 | 快速上线，无运维负担 |
| Frappe Cloud 托管（专属） | $100-500 | ¥8,400-42,000 | 官方托管 + SLA |

**推荐方案**：
- **POC 阶段**：本地 Docker 或最低配云 VPS（¥1,700-2,500/年）。
- **试运行阶段**：标准云 VPS 自部署（¥4,200-8,400/年）或 Frappe Cloud 共享（¥2,100/年）。
- **正式运行**：根据用户量和数据量选择，预计与现有云服务成本相当或略高。

### 2.3 实施与配置成本

| 实施方式 | 费用区间 | 周期 | 说明 |
|----------|----------|------|------|
| 内部团队 DIY | ¥15,000-70,000（人力时间成本） | 1-3 个月 | 需有 Python/DevOps 能力 |
| 国内 ERPNext 合作伙伴 | ¥70,000-700,000+ | 2-6 个月 | 含需求分析、配置、培训、数据迁移 |
| 混合模式（内部 + 外部顾问） | ¥35,000-150,000 | 2-4 个月 | 内部主导，外部顾问按天支持 |

**推荐方案**：
- POC 阶段完全内部 DIY，成本可控。
- 正式推广时视复杂度决定是否引入外部实施顾问。

### 2.4 培训成本

| 培训对象 | 方式 | 预估成本 |
|----------|------|----------|
| IT 运维人员 | 官方文档 + 社区 + 可能的认证培训 | ¥0-5,000 |
| 财务人员（核心用户） | 内部培训 + ERPNext 官方教程 | ¥0（内部时间成本） |
| 全体相关业务人员 | 分批次培训、操作手册、视频 | ¥0-10,000（如需外部培训师则更高） |

### 2.5 持续维护与升级成本

| 项目 | 年成本 | 说明 |
|------|--------|------|
| 服务器/云资源 | ¥2,000-10,000 | 与现有云服务相当 |
| 备份与监控 | ¥0-2,000 | 自建免费，托管服务已含 |
| 版本升级 | ¥0-10,000 | 内部升级免费，外包按次收费 |
| 安全补丁 | ¥0 | 社区版本需自行跟进 |
| 社区支持 | ¥0 | 论坛、GitHub、文档 |
| 商业支持（可选） | ¥0-35,000/年 | Frappe 或合作伙伴支持合同 |

## 4. 成本收益结论

### 4.1 直接成本对比

| 维度 | 结论 |
|------|------|
| 软件许可费 | ERPNext 可省掉用友许可费，但金额不大（¥2,500-数万/年），不是主要收益来源 |
| 服务器成本 | 自部署 ERPNext 与现有云服务成本相当，不会显著增加 |
| 实施成本 | POC 阶段 DIY 成本低；正式推广需一次性投入 ¥3-15 万 |
| 3 年 TCO | 保守方案与现状持平；中等方案前期投入高但长期可避免自研巨额成本 |

### 4.2 核心收益不在"省钱"

框架文档已明确：降本项中"减少用友费用"是**待验证方向**，不能直接承诺。

**真正的收益是避免隐性成本膨胀**：

| 收益项 | 说明 | 量化难度 |
|--------|------|----------|
| 停止自研完整 ERP 核心 | 避免继续扩展财务/库存/采购/销售引擎的长期投入 | 高（机会成本，待补实际人力成本） |
| 减少 Excel 人工汇总 | 财务人员从复制粘贴中解放 | 中（可估算工时） |
| 降低多系统对账成本 | 数据统一后，月度对账时间大幅缩短 | 中 |
| 统一数据源 | 管理层看同一套数字，决策效率提升 | 低（难以量化） |
| 库存/生产联动 | 减少库存积压和缺货损失 | 中 |

### 4.3 管理层汇报口径

**可以说的**：
- "ERPNext 软件许可费为 0，3 年基础设施成本与现状基本持平。"
- "主要收益不是省掉用友的几万块，而是避免 Workspace 继续扩展成完整 ERP 带来的长期开发和维护成本。"
- "POC 验证通过后，再评估是否逐步替代用友；替代用友的财务收益需要在 POC 完成后才能量化。"

**不能说的**：
- "上 ERPNext 每年可以省 X 万元。"（POC 未完成前无法确认）
- "用友软件费一定不需要了。"

## 5. 数据缺口与后续补充

| 数据项 | 状态 | 影响 |
|--------|------|------|
| 用友 exact 年费 | ❌ 只有区间（¥2,500-数万） | TCO 精度受限，建议财务提供精确数字 |
| 服务器 exact 年费 | ❌ 只有区间（数千-1万） | 同上 |
| 财务人员工时成本 | ❌ 未提供 | 无法量化 Excel 人工汇总的隐性成本 |
| 外包开发历史成本 | ❌ 无 | 无法对比自研 vs 引入 ERP 的机会成本 |
| 用友扩展报价 | ❌ 无 | 如果未来需要多公司/多模块，用友费用增长曲线未知 |

> ⚠️ **预算提示**：本 TCO 模型中的大额自研成本（情景 C）为按人力投入的情景估算，非确定事实。在补充实际用友 exact 年费、服务器 exact 年费、开发人力成本后，方可用于正式预算审批。
>
> **建议**：财务部门补充以下数据，使 TCO 模型可用于正式预算：
> - 用友 exact 年费
> - 服务器 exact 年费
> - 财务团队月度跨系统对账/汇总的工时估算
> - 开发团队历史/当前人力成本
