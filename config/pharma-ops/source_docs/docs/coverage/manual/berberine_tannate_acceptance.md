## 业务确认原因

鞣酸小檗碱片的三阶段页面都能访问，组件映射也不缺失；但含量项的真实业务表格尚未结构化完整。当前 `UV-对照法` 只有通用字段骨架，且 `uvReference` 组件仍是通用字段表渲染，不是按 MD 复刻的 UV 对照法表格。

这会导致页面“看起来有字段”，但用户按 MD 抽查时，称重、吸光度、重铬酸钾换算、10.69 系数、平均含量和 RD 的业务链路无法确认。

### 2. UV 对照法公式不是该产品的确认公式

MD 含量表格要求：

- 对照称样：对照 1/2 毛重、皮重、净重。
- 样品称样：样 1/2 毛重、皮重、净重。
- 对照品计算：波长、空白 OD、对照吸光值、g/OD、平均、RD≤1.0%。
- 供试品计算：小檗碱重量 = 重铬酸钾重量 × 供试品溶液吸收度 / 标准品溶液吸收度 / 10.69。
- 含量计算：样 1/2 含量、平均含量、RD≤2.0%。

当前方法问题：

- `config/methods/uv.yaml` 的 `UV-对照法` 是通用公式，并带有“品种特殊系数需调整”的 note。
- 当前字段用 `对照含量` 表达通用对照品含量，但 MD 真正使用的是重铬酸钾基准物称重和 10.69 换算系数。
- `static/js/components/assay/uv_reference.js` 目前调用 `genericFieldsTable(section)`，尚未按 MD 表格布局渲染。

## 进入 accepted 条件

满足以下条件后，鞣酸小檗碱片可由 `ready_for_acceptance_candidate` 进入 `accepted`：

1. 用户确认 `UV-对照法` 当前通用字段表是否可接受，或要求继续复刻为专用表格。

5. 重跑自动审计仍保持 MD 质量通过、`gaps: []`、组件 0 stub、business blocker/warning 为 0。
