# 编码管理页面优化清单

## Bug 修复

### 1. 员工信息/岗位信息 Tab 取消按钮无效
**文件**: `app/hr/EmployeeTab.tsx:216` / `app/hr/PositionTab.tsx:193`
**问题**: `onCancel` 只清除了编辑单元格，没退出编辑模式
**修改**:
```tsx
// 从
onCancel={() => { setEditingCell(null); }}
// 改为
onCancel={() => { setEditingCell(null); setEditMode(false); }}
```
两个文件同样修改。

## 功能优化（app/hr/page.tsx 中的 CodeTab / CodesTab）

### 2. 编辑模式去掉白边，保持紧凑宽度
**文件**: `app/hr/page.tsx`
**问题**: 编辑模式下 input 和删除按钮导致列宽变宽，与非编辑模式不一致
**方案**: 统一编号列和名称列的宽度，删除按钮不占额外宽度（用绝对定位或减小间距）
- 编号输入框 `w-16` 保持不变
- 名称输入框 `w-32` → 改为和显示时一致的宽度
- 删除按钮 `×` 不撑开列宽（可考虑编辑模式下整行hover显示删除按钮在右侧）

### 3. 再点已选部门 → 取消选择
**文件**: `app/hr/page.tsx` CodesTab
**修改**: `onSelect` 逻辑改为 toggle
```tsx
onSelect={(code) => { setSelectedDept(prev => prev === code ? null : code); }}
```

### 4. 点岗位名称 → 显示所属部门
**文件**: `app/hr/page.tsx` CodeTab
**修改**: 当 `type === "position"` 时，点击名称不再弹出人员列表，而是弹出新弹窗显示"该岗位属于哪些部门"
- 新增弹窗状态 `positionDeptModal: { open: boolean; code: string; name: string; departments: string[] } | null`
- 点击岗位名称时，查询 DepartmentPosition + Department 获取关联部门列表
- 弹窗只显示**部门名称**即可
- 人数列的点击保持原样（人员名单）

### 5. 选中部门模式下新增岗位 → 自动关联
**文件**: `app/hr/page.tsx` CodeTab + `app/api/admin/position-codes/route.ts`
**修改**:
- CodeTab 的 `handleAdd` 在有 `departmentCode` 时，新增岗位后自动调用 API 创建 DepartmentPosition 关联
- 或者改 position-codes API 的 PUT，接收可选 `departmentCode` 参数，创建岗位时同时建立关联

**推荐方案**: position-codes API 的 PUT 接收可选 `departmentCode` 参数，后端在一个事务里：
1. 创建/更新 Position 记录
2. 如果有 `departmentCode`，再创建 DepartmentPosition 关联

前端只需发一次请求，携带 `departmentCode` 参数即可。

### 6. 新增部门 → 纯新增，不自动关联岗位
**无需修改**，当前行为已符合需求。
